# План питания · 7 дней

Однофайловое веб-приложение для планирования питания на двоих (LP/IS) с автоматическим расчётом КБЖУ, закупок и заготовок.

Файл: `index.html` — всё в одном (HTML + CSS + JS, без сборки).

## Запуск

Открыть `index.html` в браузере. Никаких зависимостей и сервера.

Состояние сохраняется в `localStorage` браузера (профили, группы закупок, отметки покупок, выбор упаковок).

## Деплой на GitHub Pages

Чтобы открыть приложение с iPhone и добавить на главный экран как нативное:

### 1. Создать репозиторий
1. Зайти на github.com → **New repository**.
2. Имя: например `meal-planner`. Public.
3. Без `README` (он уже есть).

### 2. Push файлов
В папке проекта (`Meal planner`) выполнить (через Git Bash или терминал):

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/<твой-username>/meal-planner.git
git push -u origin main
```

### 3. Включить GitHub Pages
1. В репозитории → **Settings** → **Pages** (слева в меню).
2. Source: **Deploy from a branch**.
3. Branch: **main**, folder: **/(root)**. Save.
4. Через 1-2 минуты страница доступна по адресу:
   `https://<твой-username>.github.io/meal-planner/`

### 4. Добавить на главный экран iPhone
1. Открыть URL в **Safari** на iPhone (важно — в Safari, не в Chrome).
2. Нажать **«Поделиться»** (квадрат со стрелкой вверх).
3. **«На экран Домой»**.
4. Подтвердить — на главном экране появится иконка с серифной «П».

При запуске с главного экрана приложение откроется в standalone-режиме (без панелей Safari) — как нативное.

### Иконки и manifest

В корне папки рядом с `index.html`:
- `manifest.webmanifest` — PWA-манифест (имя, цвета, иконки)
- `icon-32.png`, `icon-152.png`, `icon-167.png`, `icon-180.png`, `icon-192.png`, `icon-512.png`, `icon-512-maskable.png`, `apple-touch-icon.png`

Все размеры под:
- iOS apple-touch-icon (152, 167, 180)
- PWA manifest (192, 512, 512-maskable)
- Browser favicon (32)

Чтобы заменить иконку — положи свой PNG (≥512×512) и пересоздай размеры через Pillow или онлайн-генератор (например, realfavicongenerator.net).


## Вкладки

| Вкладка | Назначение |
|---------|------------|
| **Меню** | План на каждый день: приёмы пищи, состав блюд, КБЖУ, дельта к цели |
| **КБЖУ** | Сводка факт vs цель: 8 карточек (ккал/Б/Ж/У × LP/IS) + дневные таблицы |
| **Закупки** | Список продуктов с граммовками, фильтры по группе/дню, выбор упаковки ВкусВилл |
| **Расчёт порций** | День → приём → блюда; деление готового веса между LP и IS |
| **Заготовки** | Что нужно приготовить/нарезать для каждой группы закупок |
| **Профиль** | Настройки LP/IS (вес, TDEE, цель, БЖУ) и группы закупок |

## Структура данных

Все данные — в `index.html`, в начале `<script>`:

### `FOOD` — база продуктов

```js
fid: {
  name, kcal, p, f, c,        // обязательные: имя и КБЖУ на 100г
  cat,                         // категория для группировки в Закупках
  cookFactor,                  // коэф варки: <1 теряет вес (мясо), >1 впитывает (крупа), 1 не меняется
  prep: {                      // рецепт заготовки (опционально)
    kind: 'cooked'|'cut'|'marinated',
    time: 40,                  // минут
    name: 'Куриная грудка',
    instruction: '...'
  },
  packs: [                     // варианты упаковок (для Закупок)
    { size: 500, label: '~500г филе' }
  ],
  fresh: true,                 // свежее мясо/рыба — распределяется по группам закупок
  perishable: true,            // зелень/свежие ягоды — то же
  lpOnly: true,                // не входит в Закупки (например, протеин)
  unit: '1 шт ~10г'            // подсказка для штук
}
```

### `DISHES` — блюда

```js
did: {
  name: 'Курица с гречкой',
  icon: '🍗',                  // опционально — иначе подбирается эвристикой по составу
  lp:      [['kurica', 170], ['grechka', 50]],   // [fid, граммы] для LP
  partner: [['kurica', 250], ['grechka', 80]],   // то же для IS
  cook: ['Гречку отварить.', 'Курицу разогреть.', '...'],

  // Опциональные флаги для Расчёта порций:
  unifiedCook: true,           // блюдо собирается одной массой (салат/запеканка) — одно поле деления
  cookedTogether: ['ovsyanka', 'kokmoloko'],  // эти ингредиенты варятся вместе — отдельная карточка для них,
                                              // остальные ингредиенты — просто кладутся в тарелку
}
```

Если ни `unifiedCook`, ни `cookedTogether` не указаны → каждый ингредиент с `cookFactor ≠ 1` получает свою карточку деления (для блюд типа «курица + гречка + салат», где компоненты готовятся отдельно).

### `MEALS` — приёмы пищи

```js
mid: {
  day: 'Пн',
  type: 'Завтрак' | 'Обед' | 'Перекус' | 'Ужин',
  train: 'pre' | 'post',       // опционально — пред-/пост-тренировочный приём
  dishes: ['did1', 'did2']
}
```

### `DAYS` — дни недели

```js
'Пн': {
  code: 'pn',
  type: 'train' | 'rest',      // тренировочный (💪) / отдых (🌿)
  label: 'Тренировка',
  meals: ['pn_b', 'pn_l', 'pn_s1', 'pn_s2', 'pn_d']
}
```

### `PROFILE` — настройки LP/IS (хранится в localStorage)

```js
{
  name: 'LP', weight: 80, tdee: 2300,
  goal: 'cut'|'maintain'|'bulk', goalKcal: -1000,
  proteinPerKg: 1.5, fatPercent: 30,
  targetMode: 'auto'|'manual',
  trainPct: 105, restPct: 100,    // для авто-режима: % от target в трен/отдых дни
  targets: { 'Пн': 1300, ... },   // для manual-режима
  deltaTol: 50,                    // ± коридор для зелёного маркера
  autoScale: true                  // автомасштабирование меню под цели
}
```

### `TRIPS` — группы закупок (хранится в localStorage)

```js
[
  { id: 'mon-tue', name: 'Пн - Вт', icon: '🛒', days: ['Пн', 'Вт'] },
  { id: 'wed-thu', name: 'Ср - Чт', icon: '🥬', days: ['Ср', 'Чт'] },
  { id: 'fri-sun', name: 'Пт - Вс', icon: '🧺', days: ['Пт', 'Сб', 'Вс'] }
]
```

## Ключевые механизмы

### Автомасштабирование меню (multi-macro scaling)

Когда `profile.autoScale = true`, граммовки блюд автоматически пересчитываются под цели профиля. Используется **раздельное масштабирование по макро-группам**:

- **P** (белковые): мясо, рыба, яйца, творог, протеин — масштаб для попадания в цель белка
- **F** (жирные): масло, орехи, семена, шоколад — для цели жира
- **C** (углеводные): крупы, фрукты, хлебцы, мёд — для цели углеводов
- **M / X** (смешанные): овощи, зелень — не масштабируются

Решается система 3×3 уравнений, чтобы факт совпал с **тремя целями БЖУ одновременно**. Коэффициенты зажимаются в `[0.3, 3.0]` — если упёрлись в границу, появляется предупреждение в Профиле.

### Группы закупок и заготовок

Каждый день недели попадает в одну группу TRIPS. Продукты с `fresh`, `perishable` или `prep` распределяются по группам **по дням использования** (закупаются ближе к моменту приготовления). Остальные (масло, мёд, орехи) — единая закупка в первой группе.

В Заготовках для каждой группы — карточка с подразделами:
- **Приготовить** (мясо, крупы — `prep.kind = 'cooked'`)
- **Нарезать** (овощи — `prep.kind = 'cut'`)
- **Маринование** (зарезервировано)

### Расчёт порций — три режима

1. **Варимая группа** (`cookedTogether`): одна карточка деления для группы (например, овсянка+молоко) + список «свободных» ингредиентов с фиксированными граммовками LP/IS.
2. **Унифицированное блюдо** (`unifiedCook`): одна общая карточка для всего блюда (салаты, заливное, перекусы).
3. **Раздельные компоненты** (без флагов): отдельная карточка деления для каждого ингредиента с `cookFactor ≠ 1` (мясо, крупы); сухие/свежие — без полей деления.

## Как менять / расширять

### Поправить граммовки/КБЖУ продукта
Найти `FOOD[fid]` в `index.html`, поменять `kcal/p/f/c` или `packs`. Меню/закупки/заготовки пересчитаются.

### Добавить новый продукт
Добавить запись в `FOOD` со всеми полями. Если нужна заготовка — добавить `prep`.

### Добавить новое блюдо
Добавить запись в `DISHES` (`name`, `lp`, `partner`, `cook`). Опционально `unifiedCook` или `cookedTogether`.

### Добавить блюдо в день
Добавить `did` в `MEALS[mid].dishes` или создать новый MEAL и добавить его в `DAYS[d].meals`.

### Поменять профиль или группы закупок
Через UI на вкладке **Профиль**. Сохраняется в localStorage. Сброс — кнопка «Сбросить...» там же.

### Если приложение «застряло» на старых данных
В DevTools → Application → Local Storage → удалить ключи: `profileV1`, `tripsV1`, `shopChecksV3`, `packPickV1`. Или нажать «Сбросить...» в Профиле.

## localStorage ключи

| Ключ | Что хранит |
|------|------------|
| `profileV1` | Настройки LP и IS |
| `tripsV1` | Группы закупок |
| `shopChecksV3` | Отметки купленного |
| `packPickV1` | Выбранные упаковки в Закупках |
